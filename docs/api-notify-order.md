# مستند API اطلاع‌رسانی سفارش (Notify Order)

## معرفی

این endpoint برای اطلاع‌رسانی به سیستم PineAI درباره سفارش‌های جدید یا به‌روزرسانی‌شده مشتریان استفاده می‌شود. پس از فراخوانی موفق، سیستم یک پیامک اطلاع‌رسانی برای مشتری ارسال می‌کند.

---

## اطلاعات Endpoint

| فیلد | مقدار |
|------|-------|
| **آدرس** | `https://api.pine-sms.ir/api/Order/notify` |
| **متد HTTP** | `POST` |
| **نوع محتوا** | `application/json` |
| **احراز هویت** | API Key (هدر `X-Api-Key`) |

---

## احراز هویت

برای استفاده از این API باید یک **API Key** معتبر داشته باشید.

کلید API باید در **هدر درخواست** به صورت زیر ارسال شود:

```
X-Api-Key: your-api-key-here
```

> ⚠️ در صورت نبود کلید یا نامعتبر بودن آن، درخواست با خطای `401 Unauthorized` رد می‌شود.

برای دریافت API Key با تیم پشتیبانی PineAI تماس بگیرید.

---

## ساختار درخواست (Request Body)

درخواست باید به صورت JSON و با فیلدهای زیر ارسال شود:

```json
{
  "customerPhoneNumber": "09123456789",
  "orderCode": "ORD-10045",
  "orderStatusCode": "SHIPPED"
}
```

### توضیح فیلدها

| فیلد | نوع | اجباری | توضیح |
|------|-----|--------|-------|
| `customerPhoneNumber` | `string` | ✅ بله | شماره موبایل مشتری (گیرنده پیامک) |
| `orderCode` | `string` | ✅ بله | کد یکتای سفارش در سیستم شما |
| `orderStatusCode` | `string` | ✅ بله | کد وضعیت جدید سفارش |

---

## ساختار پاسخ (Response)

### پاسخ موفق — `200 OK`

```json
{
  "success": true,
  "message": "عملیات با موفقیت انجام شد",
  "isNewCustomer": false,
  "isNewOrder": true,
  "notificationSent": true
}
```

### توضیح فیلدهای پاسخ

| فیلد | نوع | توضیح |
|------|-----|-------|
| `success` | `boolean` | نتیجه کلی عملیات (`true` = موفق) |
| `message` | `string` | پیام توضیحی درباره نتیجه عملیات |
| `isNewCustomer` | `boolean` | آیا مشتری برای اول بار در سیستم ثبت شد؟ |
| `isNewOrder` | `boolean` | آیا این سفارش جدید بود (نه به‌روزرسانی)؟ |
| `notificationSent` | `boolean` | آیا پیامک اطلاع‌رسانی به مشتری ارسال شد؟ |

### پاسخ ناموفق — `400 Bad Request`

```json
{
  "message": "توضیح خطا"
}
```

### پاسخ احراز هویت ناموفق — `401 Unauthorized`

در صورت نبود یا نامعتبر بودن API Key این پاسخ برگردانده می‌شود.


## کدهای وضعیت HTTP

| کد | معنا |
|----|------|
| `200 OK` | درخواست موفق بود |
| `400 Bad Request` | داده‌های ورودی معتبر نیستند یا خطایی در پردازش رخ داده |
| `401 Unauthorized` | API Key ارسال نشده یا نامعتبر است |

---

## نکات مهم

- مقدار `customerPhoneNumber` باید شماره موبایل ایرانی معتبر باشد تا پیامک به درستی ارسال شود.
- مقدار `orderStatusCode` باید با کدهای وضعیت تعریف‌شده در سیستم PineAI مطابقت داشته باشد.
